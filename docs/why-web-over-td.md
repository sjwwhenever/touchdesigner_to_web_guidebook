# 为什么做这个项目

**English:** [why-web-over-td.en.md](./why-web-over-td.en.md)

---

## 一个常见的场景

你在学校或工作室被推荐学 TouchDesigner。老师说它「上手快」「不用写代码」「艺术家都用它」。这些话**都没错**——但它们只是故事的一半。

另一半是：

- **TD 是商业软件**，免费版有分辨率和功能限制，正版年付并不便宜；
- **TD 作品分发很难**——你的朋友要装 TD 才能打开 `.toe` 文件，网页就是一条链接；
- **TD 的技能比较难迁移到别的工作**，而 JavaScript / TypeScript 是整个互联网行业的通用语；
- **节点编辑器看似简单，复杂起来后调试比代码更头疼**（有用过 TD 的人会懂）。

更关键的是：**2026 年了，写几行 JavaScript 的门槛已经不是 2015 年那样了。** AI 可以帮你写 90% 的样板代码，你只需要知道「该用哪个库」和「大致的工作原理」——而这两件事，正是本项目想给你的。

## 我们不是在黑 TD

认真说：TD 在以下场景里依然非常好用——

- 需要和硬件（摄像机、LED 控制器、传感器阵列、DMX 灯光）深度集成；
- 现场演出 / 装置艺术，要求低延迟、稳定、可视化调试；
- 团队中有非程序员协作者，节点图是共享语言；
- 需要 NDI、Kinect、RealSense 等专业流媒体设备支持。

这些场景里 TD 依然是最佳选择之一，**本项目不会试图说服你别用 TD**。

但如果你的需求是：

- 想把作品在浏览器里分享，或嵌入博客/作品集；
- 想和 Web 前端工程师合作；
- 想跨平台（手机、平板、PC）都能运行；
- 你还在学习阶段，想学一门迁移性更强的技能——

那么 Web 是一个值得认真考虑的选项。

## Web 栈的学习曲线真的陡吗？

十年前的答案是「有点」。今天的答案是「比想象中平缓得多」。

原因：

1. **AI 辅助**：你描述要什么，AI 写 90% 的代码。你的时间花在**调教和理解**上，而不是记 API。
2. **浏览器原生能力爆炸**：Web Audio、WebGL / WebGPU、WebMIDI、WebSocket、getUserMedia……这些曾经需要插件的东西，现在都是原生 API。
3. **零安装**：你只需要一个浏览器和一个文本编辑器。Demo 1 连 npm 都不需要。

但 AI 的前提是**你得告诉它用对的库**。这就是为什么我们做 [映射表](./mapping-table.md) 和 [AI 提示词备忘单](./ai-prompt-cheatsheet.md)。

## 我们的主张

> **不要因为怕代码而选 TD，也不要因为酷炫而选 TD。要根据你的分发目标、协作对象、学习目标来选。**

如果看完三个 demo 你觉得「Web 也挺顺手的」，那我们的目的就达到了。如果你看完依然觉得 TD 更适合你，**那也是对的选择**——至少你现在是基于信息做的决定，而不是基于老师一句话。
