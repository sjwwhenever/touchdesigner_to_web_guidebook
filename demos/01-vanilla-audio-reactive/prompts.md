# Demo 1 的 AI 提示词

这个 demo 本身就是用下面这个 prompt 生成的第一版 + 几次微调产生的。抄这个 prompt，把效果描述换成你自己的 TD 作品。

## 原始 Prompt

```
我要把我在 TouchDesigner 里的一个效果迁移到浏览器。

TD 的节点组合是：
  Audio Device In CHOP → Audio Spectrum CHOP → 取前 16 个 bin 算平均
  → 驱动一个 Circle SOP 的半径
  + 把完整频谱扔给一个 Replicator COMP 做一圈辐射线
效果是：
  一个圆的半径随麦克风低频能量呼吸，周围一圈辐射线的长度随频谱各 bin 变化。

请用 p5.js + Web Audio API 在浏览器里实现。

约束：
- 语言：JS
- 构建：不要构建工具，单 HTML 文件，从 jsdelivr CDN 加载 p5
- 目标浏览器：最新 Chrome
- 只给我一个 index.html
- 必须有一个「Click to start」按钮触发麦克风权限（浏览器要求用户手势）
- 代码要有注释，每段注释说明这对应 TD 的哪个节点
- 加一行指数平滑（等效 TD 的 Lag CHOP）让低频值不抖
- 总代码控制在 150 行以内
```

## 后续微调 Prompt 示例

如果 AI 第一版跑起来是下面这样，你可以继续跟它对话：

```
第一版跑起来了，但有两个问题：
1. 低频段 0-16 bin 实际覆盖的频率太窄（~700Hz 以下），低频呼吸感不明显。
   请把「低频」改为 0-32 bin。
2. 圆心太小，被辐射线盖住了。请让中心圆的基础半径至少是画布短边的 12%。

只改这两处，别动别的。
```

```
辐射线现在颜色和中心圆差不多，看不出来。请让辐射线的 hue 随 i（bin 索引）变化，
每根线差 4 度色相，这样能看到彩虹圈。
```

## 为什么这个 prompt 有效

1. **给出了完整的 TD 节点链**：AI 不用猜你想要什么「音频反应」。
2. **指定了最终效果**：一句人话描述，AI 可以对齐。
3. **禁用了构建工具**：AI 不会突然给你甩一个 Vite 项目。
4. **要求对照注释**：产出的代码自带教学价值。
5. **明确行数上限**：防止 AI 写得过于啰嗦。

## 把这个模版用到你自己的作品

替换这几个字段：

```
TD 节点组合：[你自己的]
效果：[你自己的]
库选择：[查 ../../docs/mapping-table.md]
构建方式：[零构建 / Vite / SvelteKit，查你自己想要哪个阶段]
```

就这么简单。
